# 隐私与权限设计

## 多租户隔离

- 所有情侣空间业务表必须包含 `couple_id`。
- 个人私密内容必须同时包含 `owner_user_id`。
- 后端禁止信任前端传入的 `couple_id`，必须从登录用户与 `couple_members` 推导。
- 查询详情时校验资源所属 `couple_id`，私密资源再校验 `owner_user_id`。

## 敏感字段

- 日记正文、未来信正文、话题答案等敏感内容使用 AES-GCM 应用层加密。
- 加密密钥来自环境变量，不进入代码仓库。
- 每条记录使用独立 nonce。

## 媒体文件

- 对象存储 Bucket 必须私有。
- 数据库只保存 object key、mime、尺寸、大小等元数据。
- 客户端读取媒体时由后端签发短期 URL。

## 管理后台边界

管理后台默认只展示元数据，不提供日记正文、照片原图、私信等私密内容的常规浏览能力。举报处理以用户主动提交的举报材料为准。
